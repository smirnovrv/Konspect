# Руководство пользователя

## Назначение

Конфигурация `Конспекты25` предназначена для ведения конспектов, хранения заметок, файлов, тем и тегов.
Основной сценарий работы: создать заметку, оформить текст, добавить теги/тему и при необходимости прикрепить файлы.

## Основные объекты

- `Заметки` — основной объект для хранения конспектов.
- `ЗаметкиФайлы` — прикрепленные файлы к заметкам.
- `Темы` — группировка и типизация заметок.
- `Тэги` — быстрые метки для фильтрации и навигации.
- `Лекции` — отдельный справочник для учета лекционных материалов.

## Быстрый старт

1. Откройте список `Заметки`.
2. Создайте новую запись.
3. Заполните тему и основной текст.
4. Добавьте теги через табличную часть `Состав`.
5. При необходимости добавьте примечания в табличной части `Примечания`.
6. Сохраните запись.

## Работа с заметками

### Создание заметки

1. Перейдите в `Заметки`.
2. Нажмите команду создания нового элемента.
3. Заполните основные поля:
   - `Тема`
   - `ОсновнойТекст`
4. Сохраните элемент.

### Редактирование заметки

1. Откройте нужную заметку из списка.
2. Внесите изменения в текст, теги или примечания.
3. Сохраните изменения.

### Табличная часть `Состав`

Используется для структурных элементов заметки:

- `Тэг` — привязка к справочнику тегов.
- `Значение` — текст/значение элемента.
- `Картинка` — изображение (если используется).
- `Ширина`, `Высота` — параметры отображения картинки.

### Табличная часть `Примечания`

Используется для дополнительных пояснений:

- `Вид` — тип примечания (из перечисления `ВидыПримечаний`).
- `Текстовка` — текст примечания.
- `id` — технический идентификатор строки.

## Работа с файлами

### Добавление файла к заметке

1. Откройте нужную заметку.
2. Используйте команду добавления файла (через форму/команду, предусмотренную в интерфейсе).
3. Выберите файл на диске.
4. Сохраните изменения.

### Что хранится в `ЗаметкиФайлы`

- файл в хранилище (`ФайлХранилище`);
- расширение файла (`Расширение`);
- владелец (`ВладелецФайла`);
- дата создания (`ДатаСоздания`);
- при необходимости строковое представление (`Base64Данные`).

## Работа с темами и тегами

### Темы

- Используйте `Темы` для группировки заметок по разделам.
- Поле `ЗаполнятьАвтоматически` может использоваться для автоподстановки/автозаполнения (в зависимости от логики формы).

### Тэги

- Создавайте теги в справочнике `Тэги`.
- Признак `Одиночный` может ограничивать повторное использование в одной заметке (в зависимости от логики модуля).
- Рекомендуется использовать короткие и понятные названия тегов.

## Перечисления в работе пользователя

- `ВидыПримечаний`: `Определение`, `Комментарий`, `Описание`, `Фрагмент`.
- `ВидыСписковHTML`: `Маркированный`, `Определений`, `Нумерованный`.
- `ЗаголовкиHTML`: `H1` ... `H6`.
- `ТипСпискаHTML`: `_1`, `A`, `I`, `_a`, `_i`.

## Рекомендации по ведению конспектов

- Одна заметка — одна законченная мысль/тема.
- Используйте теги для повторного поиска по смыслу.
- В `Примечания` выносите определения и короткие пояснения.
- Крупные файлы храните как вложения, а в тексте давайте краткий контекст.
- Поддерживайте единый стиль заголовков и списков.

## Частые действия

- Найти заметку: используйте список `Заметки` и фильтрацию по теме/тегам.
- Обновить материал лекции: откройте заметку, добавьте новые пункты и сохраните.
- Прикрепить источник: добавьте файл в `ЗаметкиФайлы`.
- Уточнить термин: добавьте строку в `Примечания` с видом `Определение`.

## Возможные проблемы и решения

- **Не сохраняется заметка:** проверьте заполнение обязательных полей и корректность значений.
- **Не добавляется файл:** проверьте доступ к файлу на диске и его размер.
- **Не находится запись:** уточните фильтры списка и правильность тегов/темы.

## Поддержка и развитие

Если требуется расширение функционала (новые команды, печатные формы, проверки заполнения), оформляйте запрос с описанием:

- что нужно сделать;
- для какого объекта (`Заметки`, `Тэги`, `Темы` и т.д.);
- ожидаемый результат для пользователя.
